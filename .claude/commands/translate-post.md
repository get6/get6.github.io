# /translate-post - 블로그 글 번역

한국어로 작성된 블로그 게시글(`blog/posts/*.md`)의 영어(en)와 일본어(ja) 번역본을 생성하는 스킬이에요.

## 실행 순서

### Step 1: 번역 대상 파일 확인

`$ARGUMENTS`로 전달된 파일 경로를 확인해요.

- 경로가 지정되면 해당 파일만 번역
- 경로가 없으면 `blog/posts/`에서 `blog/posts/en/`과 `blog/posts/ja/`에 대응하는 번역본이 없는 파일을 모두 찾아서 번역

### Step 2: 기존 번역 여부 확인

각 한국어 파일에 대해 동일한 파일명이 `blog/posts/en/`과 `blog/posts/ja/`에 존재하는지 확인해요.

- 영어 번역본: `blog/posts/en/{filename}.md`
- 일본어 번역본: `blog/posts/ja/{filename}.md`

이미 번역본이 있으면 건너뛰어요.

### Step 3: 번역 실행

한국어 원본을 읽고 영어와 일본어 번역본을 생성해요.

**유지해야 하는 것:**

- frontmatter 구조 (date, tags는 동일하게 유지)
- 대표 이미지 마크다운 (URL, alt text 그대로 유지)
- 본문의 마크다운 구조 (헤딩 계층, 리스트, 코드 블록, 표, 인용문)
- 마지막 인용문 (영어 명언이므로 그대로 유지)

**변환해야 하는 것:**

- `title`: 각 언어로 자연스럽게 번역
- 본문 텍스트: 각 언어의 뉘앙스에 맞게 번역

---

## 언어별 번역 규칙

### 영어 (en)

**톤**: 간결하고 직설적인 블로그 톤. 친근하면서도 전문적.

- 짧은 문장 선호, 능동태 사용
- 한국 고유 맥락은 독자가 이해할 수 있도록 보충 설명 또는 등가물 병기
  - 예: "다트(DART)" → "public financial filings (like DART in Korea or SEC filings in the US)"
  - 예: 한국 원화 금액 → 달러 환산 병기
- 한국어의 "~요" 체 → 영어에서는 2인칭("you") 직접 호칭으로 친근함 유지
- 불필요한 수동태, 형식적 표현 지양

**나쁜 예:**

```
It is recommended that one should consider...
In this article, we will explore...
```

**좋은 예:**

```
Consider using...
Here's what I learned...
```

### 일본어 (ja)

**톤**: 정중하면서도 읽기 쉬운 「です・ます」체. 한국어의 "~요" 체와 유사한 친근함.

- 「です・ます」체를 일관되게 사용
- 한국 고유 맥락은 일본 등가물로 현지화
  - 예: "다트(DART)" → "EDINET" 또는 일본 독자가 이해할 수 있는 표현
  - 예: 한국 원화 금액 → 엔화 환산 병기
- 지나치게 격식체(である調)는 사용하지 않음
- 카타카나 표기는 일반적인 관례를 따름

**나쁜 예:**

```
～について論じるものである。
～と言えるであろう。
```

**좋은 예:**

```
～について整理してみます。
～と言えますよね。
```

## 번역 품질 체크리스트

각 번역본에 대해 확인:

- [ ] frontmatter의 date와 tags가 원본과 동일한가?
- [ ] title이 해당 언어로 자연스럽게 번역되었는가?
- [ ] 대표 이미지 마크다운이 그대로 유지되었는가?
- [ ] 헤딩 구조(##, ###)가 원본과 동일한가?
- [ ] 코드 블록, 표, 리스트 구조가 유지되었는가?
- [ ] 마지막 인용문(영어 명언)이 그대로 유지되었는가?
- [ ] 한국 고유 맥락이 해당 언어 독자에게 이해 가능하게 현지화되었는가?
- [ ] 번역투가 아닌 해당 언어 네이티브의 자연스러운 문체인가?

---

## 출력 위치

- 영어: `blog/posts/en/{원본 파일명}.md`
- 일본어: `blog/posts/ja/{원본 파일명}.md`

---

## 주의사항

- 원본 한국어 파일은 절대 수정하지 않기
- 번역본이 이미 존재하면 건너뛰기 (덮어쓰지 않기)
- 사용자가 제공하지 않은 내용을 임의로 추가하지 않기
- 의역은 허용하되, 원본에 없는 정보를 추가하지 않기
- 이모지 사용하지 않기
