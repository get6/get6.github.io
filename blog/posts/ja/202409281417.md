この記事は、事業をするために企画しろという話が多かったので、企画とは何か、企画書はどのように作成すべきかをインターネットで調べてまとめたものです。

**企画書**とは、企業で行う仕事を文書に残し、具体的な計画を整理して他の人に共有するために作成するものです。

**企画**とは、ある問題点や課題に対して起きていることや現象を分析し、その問題点が解決する価値があるか、この問題が解決されたらどのような影響があるか、この問題を解決する方法はあるかといった項目を、問題を解決するために設定した仮説を解決方法によって期待される利益と結果を、実行する前にあらかじめ予想してみることです。期待以上に利益が大きかったり結果が良ければ、この仕事をより情熱的に進められ、一緒にやる仲間を集めることができます。

**企画のステップ**

1. 問題点および改善案の発生
2. 問題および課題の確認
3. 問題および課題の検証／現状分析
4. 解決方法の導出
5. 解決のための具体的な実行計画
6. 企画書の作成

## 企画書を作成する前に必要な事前準備

| WHY      | なぜ企画を進め、その目標は何か？<br/>企画を通じて期待することは何か？ | 企画の背景<br/>企画目標 |
| WHAT     | 企画の内容は何か？                                                    | 企画の内容              |
| HOW      | 企画はどのような方法で進めるか？                                      | 企画方法                |

| WHEN     | 企画の実施時期はいつで、期間はどれくらいか？                          | 企画時期<br/>企画期間   |
| WHERE    | 企画はどこで進めるか？                                                | 企画場所                |
| HOW MUCH | 企画を進めるためにどの程度の予算が必要か？                            | 企画予算                |

| 構成     | 説明                               | 目的                                     |
| -------- | ---------------------------------- | ---------------------------------------- |
| WHO      | 誰が、誰のための企画か？           | 実行者、関係者                           |
| WHY      | なぜこの企画が必要か？             | 企画の意図、理由、背景                   |
| WHAT     | 何のための企画か？                 | 企画のテーマ、内容                       |
| WHERE    | どこで実施するか？                 | 地理的・自然的な環境場所                 |
| HOW      | 企画をどのように実践・推進するか？ | 方法、手順、ツール                       |
| HOW MUCH | どれくらいの費用がかかるか？       | 予算、損益計算                           |
| WHEN     | いつから開始するか？               | どのスケジュールで実行？タイミング、期間 |
| HOW LONG | いつから持続するか？               | 進行期間、活性期間                       |

**企画の進行順序・方法**
企画書を作成する前に、まず企画を整理する必要があります。プロセスは以下の通りか、少し異なる場合もあります。

1. 課題の認識
   課題の方向性を見つける、現状分析、原因分析、条件検討、要件、ニーズ分析などを行う段階
2. 情報の収集および分析
   必要な情報を収集し分析する段階
3. 発想・構想
   必要なアイデアを創出し解決方法を見つけ出す段階
4. 企画書作成段階
   文書を作成し表現する段階。文書の種類は企画書、提案書、報告書、稟議書などの形式で作成
5. 報告段階
   発表は企画の類型によって異なる。表現の方法も報告、説得、情報提供などの形式で行われる

**企画書作成時の注意事項**

1. 最後に結論を正確に記載し、全体の理解を助けられるようもう一度整理する必要があります
2. デザインや内容がぎっしり詰まって見えてもいけないし、少なすぎてもいけません
3. 散漫さよりも簡潔さを優先し、レイアウトを洗練され統一されたものにする必要があります
4. 表や画像などを自作するか、全体的な雰囲気に合うように使用する必要があります
5. 全体的な方向性、企画書で伝えたいことが企画の意図と一致する必要があります
6. 企画書を読む人、依頼者、担当者の立場で作成し、理解しやすいものにする必要があります
7. 予想される質問や疑問点にあらかじめ回答を用意します
8. 個人的な経験、考え、主観的な意見ではなく、データに基づいて説得する必要があります

## 企画書の基本構成

**表紙部**

| 構成            | 説明                                                                                        |
| --------------- | ------------------------------------------------------------------------------------------- |
| 名称<br/>起案者 | マーケティング企画書／提案書／戦略報告書<br/>代表者／個人名／チーム名／チームメンバー名など |
| はじめに        | 挨拶／序文／企画の経緯などを簡潔にまとめて記載                                              |
| 目次            | 章、節などで企画書の順序を明示                                                              |

**導入部**

| 構成                  | 説明                                                                                                |
| --------------------- | --------------------------------------------------------------------------------------------------- |
| 企画目的<br/>企画背景 | 経営方針／経営戦略／経営政策および手続きなどの適切な構成<br/>企画の目的と目標、理由などを記述する。 |
| 期待効果              | 挨拶／序文／企画の経緯などを簡潔にまとめて記載                                                      |
| 目次                  | 実行結果の期待効果および予測効果、影響などを記述                                                    |

**本論部**

| 構成     | 説明                                                                                                    |
| -------- | ------------------------------------------------------------------------------------------------------- |
| 企画内容 | 解決方法／構想案の提示（構想案が多い場合は長所・短所の比較）<br/>企画実行のための戦略／方法／手段の提示 |
| 実施計画 | 実施のための組織化計画／詳細作業計画／スケジュール計画などを整理                                        |
| 予算     | 企画予算／所要経費の算出／資金調達方法                                                                  |

**補助部**

| 構成                 | 説明                                                 |
| -------------------- | ---------------------------------------------------- |
| 環境変数に対する対策 | 企画推進時の問題点<br/>問題に対する対策の策定        |
| 関係資料             | 企画立案のための調査資料および統計資料／参考文献など |

## 企画書作成法 実践3ステップ

1. 設定 - 事前の下書きを描こう
2. 構成 - 基本の骨格を組み立てよう
3. ページ作成 - 各ページの本文を書こう

### 設定 - 事前の下書きを描く

1. 企画書の作成方向の設定
2. 企画書のオプション設定
3. 企画書の説得方向の設定

**企画書の作成方向の設定**
どのような形式で進むべきか、あらかじめ方向を考えて決めることです。
自分や会社の状況だけで判断して企画書を作成せず、周囲の状況（業界、市場の雰囲気など）と企画書を読む人のタイプや状況、競争力の有無を把握する必要があります。
もし企画書が不十分であれば、そのままにせず改善点や補える部分を企画書に記載すべきです。

**企画書のオプション設定**
環境面（財政状態、ビジョン、方向性、アイテムと外部状況など）の整理と必要な前提条件（企画実行に必要なリソース）を確認すること。
確認すべきことと核心的な内容を確認し、企画書に盛り込めるよう事前に整理と定義をしておく必要があります。

**企画書の説得方向の設定**
すでに90%程度関心があって何を言っても説得される人ではなく、自分の企画に関心のない人、もともと否定的な人を説得する企画書を作ることが大切です。
当然、説得するために作るものなので、説得できるように作らなければなりません。説得されもしないのに、こんなに一生懸命作る必要があるでしょうか？今やるべき他の仕事がたくさんあるのに。

企画書作成の方向を決めるために確認すべき**3つのこと**があります。

1. 企画書の説得タイプを決める
2. 企画書で打ち出すアイテムを点検する
3. 説得の訴求を検討する

**企画書の説得タイプを決める**
読み手が企画書のどの部分に惹かれるかということです。
「これが欲しかったんだ…！」「個人的にはそこまで惹かれないけど、そういう人は多そうだね。」「お金になりそうだ。」「他の人は食いつきそうだ。」「斬新だね。」

どうやって説得したい人をこの企画書に引き込むかを考えてみてください。

**企画書で打ち出すアイテムを点検する**
説得するタイプが決まったら、アイテムについても点検します。
自分のアイテムのどの機能、UIや強み、あるいはアイデアが説得要素として使えるかを考える必要があります。
深く考えずに一つの機能を選んで書いていて「これはあまり響かなそうだ…」と思ったら、変えるべきです。

**説得の訴求を検討する**
説得される人にどのような感情を感じてもらうか、雰囲気を決めます。
恐怖、警戒心、期待感、魅力など、決められた訴求によって伝えるTone & Mannerが変わるべきです。

### 構成 - 基本の骨格を組み立てよう

直接的な骨格とは目次のことですが、その骨格を構成する前に企画書の展開（内容を進展させること）を構成する必要があります。そのために頭括式と尾括式の表現を知っておく必要があります。

小説の起承転結と似ていますが、勝負どころをどこに置くかを決める必要があります。
頭括式は最初から結論を提示し、なぜそのような結論なのかを説明する過程を経ます。
尾括式は問題提起から解決策という順序で、最後に自社のアイテムを紹介する過程を指します。

展開構図を設定したら、あとは目次を作ればよいです。目次は道しるべのようなものです。
どこに向かうべきかを教えてくれるので、目次がうまくできていればナビゲーションのように目的地にたどり着けます。
何らかの理由で目次を企画書に入れない場合でも、柔軟に対処するために作っておく必要があります。

内容は英語でContent
目次はContents、Table of contents、List of contents

目次を適切に設定するのは難しいです。目次の設定を間違えるとどうなるか、企画書を作成中にこのような状況に遭遇することがあります。

1. 全体像が把握できない。結局何を言いたいのか分からない
   目次を設定しないと起承転結を作るのが難しくなります
2. 全部直すか、作り直さなければならない
   疎結合ではなく密結合なので、現在のスライドを変えると前のスライドと話が合わなくなったり、つながる感じがしなくなる可能性があります。つながる感じは段階的に項目ごとに進行するという感覚で代替できます
3. 追加のスライドや修正事項への対応が難しくなる
   途中で追加要件が入ってきた場合、目次がきちんと整理されていないと、どこに入れればよいか曖昧になることがあります

**目次をうまく作る方法**
起承転結を導くための内容が少なすぎても問題ですが、内容が多すぎて書くことが多くても問題です。
データがなければ考察と調査を、データが多ければ枝刈りをして重要な内容を選別します。

1. 大きな塊に分けてみる
2. 部分集合と共通部分を選び出す
3. 順序付けする

**大きな塊に分けてみる**
塊同士が同じくらいの大きさに感じられる必要があります。
ソフトウェア開発で分けると、企画、開発、デザイン、テスト、デプロイ、運用などの単位で見ることができます。
こうした塊レベルの分類はAIに質問したり、検索して解決すると役立ちます。

**部分集合と共通部分を選び出す**
集合とは、同じ内容のものがまとまっていることです。
すでにおおまかに説明した内容を後でまた詳しく説明するほど退屈なことはありません。
こうした部分的な重複を解決するには、片方に寄せます。

**順序付けする**
建物の模型のように一度に視覚的に複数を見せることはできず、一度に説明するのも難しいです。
内容がつながりながら一つずつ順番に説明する必要があります。
一枚一枚話す順序とつながる順序が流れに沿って連結されていないと不自然で、相手も理解しにくくなります。
シャワー後に服を着るとき（シャワーの流れ、服を着る前のドライヤーとスキンケア、服を着る順序）に順序があるように、決められた基準を持って上下関係が崩れないように決める必要があります。

### ページ作成 - 各ページの本文を書こう

目次を決めたので、企画の難しい部分を一つ解決しました。
もちろんページ作成にも順序があります。

1. Pagenation
2. 核心ワーディングの作成
3. ページの要素を理解する
4. ページの本文を書く

**Pagenation**
大きな塊ごとにどれくらいのページを割り当てるべきかを区分することです。大きな塊が多ければページ数も多くなります。大きな塊を1ページに収めようとすると、1枚が複雑になったりテキストが多くなってピンとこない場合があります。重要でないところに多くのページを割り当てると、企画の意図から逸れてしまいます。ページネーションはバランスゲームです。

**核心ワーディングの作成**
各ページで何を伝えたいのか、簡単なフレーズで整理できている必要があります。
これを軽視すると、簡単に言えることを難しく言ったり、重要でないことを重要そうに言ってしまうことがあります。

核心ワーディングを作ることで得られるメリットがあります。

1. ページごとに何を書くべきかが明確になります
2. 各ページから抽出したワーディングをすべてつなげると、企画書で伝えたいストーリーがすぐに見えます
3. 抽出したワーディングをつなげてみると、どこが弱いかが分かり、不足箇所を補強できます
4. 核心ワーディングを作ってつなげると、企画書の要約が楽になります

#### ページの要素を理解する

各ページに何を入れればよいか悩む部分です。
ページを構成する要素は大きく文章、図、表、インフォグラフィックです。

**文章**
文章が一つもないページはあるでしょうか？あるとすれば、それは核心ページではないでしょう。何の文章を書くか決めるのが難しければ、以下から選べばよいです。

1. 核心ワーディングを含むテーマを伝える文章
2. テーマを説明する文章
3. 単純な情報を明示する注釈のような文章

**表**
載せるべき内容が多すぎるとき、見やすいガイドで整列するものです。
ページに表だけをぽつんと置いてはいけません。企画書の表は**確認用**であり、説得対象が**分析する用**ではありません。
表から分かる内容を要約して文章で表現する必要があります。

**図**
意味のない図は避け、ページと関連する内容の画像を挿入すると理解を助けられます。
図は効果が大きいのでよく使われますが、各ページとの関連性も考慮する必要があります。

**インフォグラフィック**
インフォグラフィックとは、情報をより分かりやすくするために視覚的に表現したものです。
ダイアグラム、グラフなどもインフォグラフィックの範囲に含まれます。
多すぎると雑然として集中を妨げるため、適切に使用する必要があります。

**ページの本文を書く**

1. 核心文章を書きます
2. 核心文章を裏付けるページの4要素の資料をすべて入れます
3. 一つずつ最低限必要なものだけ残し、残りは外します
4. 内容が見やすいようにデザインを編集します

一つのページを10分ずつ読むのか、人々の集中力がどれくらい持つのかを考慮しながら企画書を作成する必要があります。熱心に見てくれても集中力は切れるでしょう。むしろページごとに1、2個の情報が整理されている方が相手への配慮になります。時間があれば[企画に関する誤解と真実、企画書作成法の基礎理論1](https://think-trigger.com/%ea%b8%b0%ed%9a%8d%ec%97%90-%ea%b4%80%ed%95%9c-%ec%98%a4%ed%95%b4%ec%99%80-%ec%a7%84%ec%8b%a4/)を読むこともおすすめします。

## 企画書の構成

1. 企画背景
2. 現状分析
3. 企画の核心／内容
4. 実行戦略／方法
5. 期待効果

**企画背景**
企画を始めた理由を整理します。個人的な問題を解決したかったのです。システムが助けてくれなければ、自分自身を変えることが難しかったです。

**現状分析**
その理由に関連する現在の状況を把握します。競合はいるか、市場規模と状況はどうか、自分と会社の状況はどうか、どれくらい実現できるかを客観的なデータに基づいて綿密に確認します。

企画が必要な理由を客観的な資料に基づいて証明（Why）し、どのようにどれだけ解決可能な解決策を紹介（So what）する必要があります。

**企画の核心／内容**
自分が考えた問題点をすべて解決してくれるスーパーアプリになってほしいです。しかし、多くのことを書くと特定の問題への集中度が下がり、一度にすべてを実行できるリソースを持っていないので、調整する必要はあります。
問題点の中から共通点をキーワードとして抽出する作業を行い、単純化する必要があります。

**実行戦略／方法**
実行はどこからどこまでを言うかによって範囲が変わります。どの段階の企画書かによっても実行戦略は変わります。IT産業では大きく予算、マーケティング計画、ソフトウェア開発計画に分けられます。
重要なのは、多くの人が採用する方法をうまく実行することと、他社と比べても遜色ないアイデアとコスト対効果を得られる創造的な方法を提示して、企画書を読む人を満足させることです。その上で消費者に集中している姿勢を見せる（What）必要があります。

**期待効果**
この企画を実行して得られる効果。ブランド確保なのか、利益創出または増加なのか、潜在顧客の確保なのか、さまざまな企画によって異なる結果を得ることができます。そして必ず進めていく中で、期待していなかったポジティブな効果が生まれるはずです。
